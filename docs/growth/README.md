# Performance & Growth Documentation

This directory is the home for personal performance, growth, and learning notes — current and future.

## Structure

```
docs/growth/
├── README.md               # This file
├── leadership-principles.md  # MongoDB Leadership Principles reference
└── cycles/                 # Per-review-cycle growth notes (one file per cycle)
```

## MongoDB Leadership Principles (Quick Reference)

- **Leaders act like owners** — Take responsibility end-to-end; act in the long-term interest of the company.
- **Leaders communicate with clarity** — Written and spoken communication is precise, structured, and audience-aware.
- **Leaders focus on outcomes** — Prioritize impact over activity; define success before starting work.
- **Leaders never stop learning and growing** — Seek feedback, reflect, and continuously improve skills and knowledge.
- **Leaders build extraordinary teams** — Hire and develop the best, and create an inclusive environment where people thrive.

## How to Use

1. After each meaningful feedback event (review, Slack thread, 1:1), run the `growth-notes` Claude skill on the thread.
2. Paste the output into a file under `cycles/` named `YYYY-QN.md` (e.g., `2025-Q3.md`).
3. Review and refine the output before committing.
4. At the start of each new cycle, create a fresh cycle file and carry forward the most important "Next cycle focus" items.
